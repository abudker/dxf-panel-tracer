**src/components/RulerOverlay.tsx** -- HTML ruler overlay at bottom of canvas:

1. Import `useAppStore` from store
2. Export named function `RulerOverlay`
3. Subscribe to `calibration` and `viewport` from store:
   ```typescript
   const calibration = useAppStore((s) => s.calibration);
   const viewport = useAppStore((s) => s.viewport);
   ```
4. If `calibration === null`, return null (ruler only shows after calibration)
5. Compute tick spacing:
   - `const { pxPerMm, unit } = calibration;`
   - `const screenPxPerMm = pxPerMm * viewport.scale;` (how many screen pixels per mm at current zoom)
   - For inches display: convert internally but label in inches. The ruler always computes in mm internally but displays the user's chosen unit.
   - Implement `pickNiceInterval(screenPxPerUnit: number): number` helper function (can be inline or module-level):
     - Define nice steps based on unit:
       - For mm: `[1, 2, 5, 10, 20, 50, 100, 200, 500, 1000]`
       - For inches: `[0.25, 0.5, 1, 2, 5, 10, 20, 50]` (in inches, converted from mm)
     - If unit is 'in': `screenPxPerUnit = screenPxPerMm * 25.4` (screen px per inch)
     - TARGET_MIN_PX = 60 (minimum screen pixels between major ticks -- ensures readability)
     - Find the smallest nice step where `step * screenPxPerUnit >= TARGET_MIN_PX`
     - Return that step (in display units: mm or inches)
   - `const majorInterval = pickNiceInterval(...)` (in display units)
   - `const majorIntervalPx = unit === 'in' ? majorInterval * 25.4 * screenPxPerMm : majorInterval * screenPxPerMm` (in screen pixels)

6. Compute visible tick range:
   - The ruler spans the full viewport width (`window.innerWidth`)
   - The world-coordinate x at the left edge of the screen: `const worldLeftX = -viewport.x / viewport.scale`
   - The world-coordinate x at the right edge: `const worldRightX = (window.innerWidth - viewport.x) / viewport.scale`
   - Convert to display units: divide by pxPerMm, then by 25.4 if inches
   - `const startUnit = Math.floor(worldLeftX / pxPerMm / (unit === 'in' ? 25.4 : 1) / majorInterval) * majorInterval`
   - `const endUnit = Math.ceil(worldRightX / pxPerMm / (unit === 'in' ? 25.4 : 1) / majorInterval) * majorInterval`
   - Generate tick positions from `startUnit` to `endUnit` in `majorInterval` steps

7. For each tick, compute its screen-x position:
   - `const worldX = tickValue * (unit === 'in' ? 25.4 : 1) * pxPerMm` (world pixels)
   - `const screenX = worldX * viewport.scale + viewport.x` (screen pixels)

8. Render the ruler strip:
   - Outer container: `fixed bottom-0 left-0 right-0 h-[32px] z-40 pointer-events-none` with `bg-[#1a1a1a]/80` (semi-transparent dark background per locked decision)
   - Add a top border: `border-t border-[#3a3a3a]`
   - For each tick, render:
     - A vertical line: `absolute` div, `w-px h-[12px] bottom-0 bg-[#666]`
     - A label: `absolute` span, `text-[10px] text-[#999]` positioned just above the tick line
     - Label format: `{value}{unit === 'mm' ? 'mm' : '"'}` (e.g., "50mm", "2"")
     - Only show label on major ticks (every tick is a major tick since we use pickNiceInterval)
   - Add 4 minor ticks between each major tick pair (subdivide into 5 segments):
     - Minor tick: `w-px h-[6px] bottom-0 bg-[#444]`
     - No label on minor ticks
   - Cap the total number of rendered ticks to avoid performance issues: if more than 200 total tick elements would be generated, skip minor ticks

9. Unit indicator: show a small label in the bottom-right corner of the ruler strip: `text-[10px] text-[#666] absolute right-[8px] top-[4px]` showing the full unit name (e.g., "millimeters" or "inches")
