Implement the two-click scale calibration flow: types, Zustand state, click handling on canvas, visual feedback (dots + dashed line), modal dialog for distance entry, toolbar calibrate button with badge, toast warning for uncalibrated drawing attempts, and scale-first enforcement.

Purpose: This is the core calibration interaction that PHOTO-04 and PHOTO-06 require. Without it, drawing tools cannot store world-coordinate geometry and DXF export would produce dimensionless output.

Output: Working calibration flow where user clicks two photo points, enters a real-world distance, and the app computes and stores pxPerMm. Drawing tools are blocked until calibration is set.

**src/store/useAppStore.ts** -- Add calibration slice to existing store:

1. Import `CalibrationState`, `CalibrationClickState`, `CalibrationUnit` from types
2. Add to AppState interface:
   - `calibration: CalibrationState | null` (null = not calibrated)
   - `calibrationClick: CalibrationClickState`
   - `toastMessage: string | null` (for scale-first warning toast)
   - `addCalibrationPoint: (p: Point) => void`
   - `confirmCalibration: (distance: number, unit: CalibrationUnit) => void`
   - `cancelCalibration: () => void`
   - `resetCalibration: () => void`
   - `setToastMessage: (msg: string | null) => void`

3. Initialize defaults: `calibration: null`, `calibrationClick: { clickPoints: [], isModalOpen: false }`, `toastMessage: null`

4. Implement `addCalibrationPoint(p)`:
   - Get current `calibrationClick` from state
   - If `clickPoints.length === 0`: set `clickPoints: [p]`
   - If `clickPoints.length === 1`: set `clickPoints: [...clickPoints, p]` AND `isModalOpen: true`
   - If `clickPoints.length >= 2`: ignore (modal is open, no more clicks)

5. Implement `confirmCalibration(distance, unit)`:
   - Get `calibrationClick.clickPoints` (must have length 2)
   - Compute pixelDist: `Math.sqrt((p2.x - p1.x)**2 + (p2.y - p1.y)**2)`
   - Convert distance to mm: `const distanceMm = unit === 'in' ? distance * 25.4 : distance`
   - Compute `pxPerMm = pixelDist / distanceMm`
   - Set `calibration: { pxPerMm, unit }`, `calibrationClick: { clickPoints: [], isModalOpen: false }`, `toolMode: 'select'`

6. Implement `cancelCalibration()`:
   - Set `calibrationClick: { clickPoints: [], isModalOpen: false }`, `toolMode: 'select'`
   - Do NOT clear existing `calibration` (preserve prior calibration if re-calibrating)

7. Implement `resetCalibration()`:
   - Set `calibration: null`, `calibrationClick: { clickPoints: [], isModalOpen: false }`

8. Modify `setToolMode`: When the requested mode is a future drawing tool (not 'select' and not 'calibrate') AND `calibration === null`, set `toastMessage: 'Set scale reference first'` and do NOT change toolMode. This implements PHOTO-06.
   - For now, since only 'select' and 'calibrate' exist, add this guard as a forward-looking check: `if (mode !== 'select' && mode !== 'calibrate' && !get().calibration)` -- this will activate when Phase 3 adds 'line' and 'arc' to ToolMode.
   - When toolMode is set to 'calibrate', also call `resetCalibration` logic inline: clear `calibrationClick` to `{ clickPoints: [], isModalOpen: false }` so re-calibration starts fresh.

   ```typescript
   const handleStageClick = (e: KonvaEventObject<MouseEvent>) => {
     const state = useAppStore.getState();
     if (state.toolMode !== 'calibrate') return;
     const stage = e.target.getStage();
     if (!stage) return;
     const pointer = stage.getPointerPosition();
     if (!pointer) return;
   ```
