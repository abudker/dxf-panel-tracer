# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — Konva handles canvas; Tailwind CSS v4 handles HTML shell |
| Icon library | Lucide (lucide-react) — tree-shakable, MIT, clean line style |
| Font | System font stack: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif |

**Rationale (source: CONTEXT.md + CLAUDE.md):** The app is canvas-centric. The HTML UI surface in Phase 1 is one floating toolbar and one drop-zone overlay. shadcn introduces a component library dependency where a few Tailwind utility classes suffice. Lucide is the standard companion for Tailwind projects and provides upload/zoom icons without custom SVGs.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside toolbar buttons |
| sm | 8px | Toolbar internal padding, badge spacing |
| md | 16px | Toolbar outer padding, drop-zone text margin |
| lg | 24px | Drop-zone vertical padding |
| xl | 32px | Drop-zone border width (visual breathing room) |
| 2xl | 48px | Drop-zone min-height anchor |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Toolbar icon buttons: minimum touch target 44px × 44px (accessibility floor, not on the 8-point scale)
- Toolbar container positioned 16px from top-left viewport edge (source: CONTEXT.md — "top-left floating overlay")

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / toolbar label | 14px | 400 (regular) | 1.5 |
| UI label / badge | 12px | 500 (medium) | 1.4 |
| Drop-zone heading | 20px | 600 (semibold) | 1.2 |
| Drop-zone subtext | 14px | 400 (regular) | 1.5 |

**Note:** Only 3 distinct sizes are used (12, 14, 20). Heading weight (600) and body weight (400) are the two declared weights. Medium (500) is used only for uppercase badge labels — not a third structural weight.

|------|-------|-------|
| Dominant (60%) | #1a1a1a | Full-viewport app background; canvas surround |
| Secondary (30%) | #2a2a2a | Floating toolbar background; tooltip surfaces |
| Accent (10%) | #3b82f6 | Upload button active state; drag-over drop-zone border; file picker button |
| Canvas surface | #ffffff | Konva stage background — distinct from app bg |
| Destructive | not applicable | No destructive actions in Phase 1 |
| Border / divider | #3a3a3a | Toolbar border, drop-zone dashed border (normal state) |
| Text primary | #f5f5f5 | Toolbar labels, drop-zone heading |
| Text secondary | #a3a3a3 | Drop-zone subtext, keyboard shortcut hints |

**Source:** CONTEXT.md decision — "Dark UI (dark gray toolbar/bg) with light canvas"

Accent reserved for:
- Upload/open-photo button (hover + active state only — idle state is secondary bg)
- Drop-zone border when a file is dragged over the window (drag-enter feedback)
- File picker `<input>` trigger button

Accent is NOT used on: toolbar icon buttons at rest, canvas elements, status text.

## Interaction Contract

### Canvas — Pan
- Cursor: `grab` at rest when photo loaded; `grabbing` during active drag
- Trigger: pointer down + drag anywhere on canvas
- Constraint: no boundary — user can pan freely (photo can go off-screen)

### Canvas — Zoom
- Trigger: scroll wheel over canvas
- Anchor point: cursor position (source: CONTEXT.md)
- Min zoom: 10% (0.1 scale factor)
- Max zoom: 1000% (10.0 scale factor)
- Zoom increment: 10% per scroll tick (multiplicative, not additive)
- Cursor: unchanged during zoom (no cursor change needed)

### Drop Zone — Drag and Drop
- Drag-enter window: drop zone border changes from `#3a3a3a` dashed to `#3b82f6` solid; heading changes to "Drop to open"
- Drag-leave window: border resets; heading resets
- Drop: load file immediately; no confirmation dialog

### Photo Display
- Initial fit: `contain` — entire photo visible in viewport at load (source: CONTEXT.md)
- EXIF rotation: applied on load so portrait phone photos are correctly oriented (ROADMAP success criterion 4)
- HiDPI: Konva pixelRatio set to `window.devicePixelRatio` (source: CONTEXT.md)

### Toolbar
- Position: fixed, 16px from top edge, 16px from left edge, z-index above canvas
- Background: `#2a2a2a` with 1px `#3a3a3a` border, 8px border-radius
- Opacity: 1.0 (not translucent — clarity over aesthetics for a precision tool)
- Phase 1 toolbar contains: "Open Photo" button only
- Button idle: bg transparent, text `#f5f5f5`, icon `#a3a3a3`
- Button hover: bg `#3a3a3a`
- Button active (click): bg `#3b82f6`, text white
