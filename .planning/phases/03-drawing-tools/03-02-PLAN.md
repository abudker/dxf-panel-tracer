Compute `snapTarget` by checking if `drawing.cursorWorld` is within snap threshold of any endpoint:

const snapTarget = useMemo(() => {
  if (!drawing.cursorWorld || (toolMode !== 'line' && toolMode !== 'arc')) return null;
  const result = snapToEndpoint(drawing.cursorWorld, segments, viewport, 10);
  return result.snapped ? result.point : null;

    ctx.beginPath();
    ctx.arc(seg.center.x, seg.center.y, seg.radius, seg.startAngle, seg.endAngle, seg.anticlockwise);
    ctx.fillStrokeShape(shape);
  }}
  stroke={seg.id === selectedId ? '#f59e0b' : '#22d3ee'}
  strokeWidth={2}
  hitStrokeWidth={20}
  fill="transparent"

**src/hooks/useDrawingKeys.ts — NEW file:**

Named export: `export function useDrawingKeys()`

Register a `window.addEventListener('keydown', handler)` in a `useEffect` with empty deps array. Clean up on unmount.

Handler logic (always read state via `useAppStore.getState()` — per established pattern to avoid stale closures):

1. **Undo:** `(e.metaKey || e.ctrlKey) && !e.shiftKey && e.key === 'z'` -> `e.preventDefault(); state.undoDraw();`
2. **Redo:** `(e.metaKey || e.ctrlKey) && e.shiftKey && (e.key === 'z' || e.key === 'Z')` -> `e.preventDefault(); state.redoDraw();`
3. **Delete selected:** `(e.key === 'Delete' || e.key === 'Backspace') && state.selectedSegmentId` -> `e.preventDefault(); state.deleteSegment(state.selectedSegmentId);`
4. **Tool shortcuts:** `e.key === 'l' || e.key === 'L'` -> `state.setToolMode('line')`. Same for `a`/`A` -> `'arc'`, `s`/`S` -> `'select'`. Only activate if no modifier keys held (no metaKey, ctrlKey, altKey) to avoid conflicts with browser shortcuts like Ctrl+S.
5. **Escape:** `e.key === 'Escape'` -> `state.clearDrawing(); state.selectSegment(null);` — cancel in-progress drawing or deselect.

Import: `useEffect` from react, `useAppStore` from store.
