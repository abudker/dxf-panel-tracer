**src/utils/geometry.ts — new file with all geometry algorithms:**

1. `circumcircle(p1: Point, p2: Point, p3: Point): CircumcircleResult` — returns `{ center: Point, radius: number, collinear: boolean }`. Use the standard circumcircle formula from the research (A, B, C, D, E, F, G computation). Collinear threshold: `Math.abs(G) < 1e-6`.

2. `arcDirectionFromThreePoints(p1: Point, p2: Point, p3: Point, center: Point): boolean` — returns `anticlockwise` boolean for `context.arc()`. Algorithm: compute angle of p3 relative to center (`a3 = Math.atan2(p3.y - center.y, p3.x - center.x)`). Compute startAngle from p1 and endAngle from p2. Normalize all angles to [0, 2*PI]. Check if a3 falls within the CCW sweep from startAngle to endAngle. If yes, return `false` (canvas CCW = `anticlockwise: true` in canvas API, but canvas draws CW by default when `anticlockwise=false`). Specifically: check if traveling counterclockwise from startAngle to endAngle passes through a3. If so, `anticlockwise = false` (draw CW sweep which is the OTHER arc). If not, `anticlockwise = true`. The key insight: `context.arc(cx, cy, r, start, end, false)` draws clockwise from start to end. We want the arc that passes through p3, so check which direction (CW or CCW in canvas terms) contains p3.

3. `getEndpoints(segments: Segment[]): Point[]` — extracts all unique endpoints. For LineSegment: start and end. For ArcSegment: p1 and p2 (the clicked start/end points, NOT center).

4. `snapToEndpoint(worldClick: Point, segments: Segment[], viewport: Viewport, thresholdScreen: number): { point: Point, snapped: boolean }` — converts threshold from screen to world pixels via `thresholdScreen / viewport.scale`. Finds nearest endpoint within threshold. Returns `{ point: snappedPoint, snapped: true }` or `{ point: worldClick, snapped: false }`. Import Viewport and Point from types, Segment from types.

5. `normalizeAngle(angle: number): number` — normalize to [0, 2*PI) range: `((angle % (2 * Math.PI)) + 2 * Math.PI) % (2 * Math.PI)`.

6. `isAngleInSweep(angle: number, start: number, end: number): boolean` — checks if normalized angle is in the counterclockwise sweep from start to end. Used internally by arcDirectionFromThreePoints.

All functions: export as named exports. Import Point, Viewport, Segment, ArcSegment, LineSegment from '../types'.

**Implement actions:**

1. `addSegment: (seg) => { ... }` — Push current `segments` to `drawHistory` (capped at 50 via `.slice(-50)`). Set `segments` to `[...segments, seg]`. Clear `drawFuture` to `[]`. Clear `drawing` to `{ clickPoints: [], cursorWorld: null }`. Clear `selectedSegmentId` to `null`.

2. `deleteSegment: (id) => { ... }` — Push current `segments` to `drawHistory` (capped at 50). Set `segments` to `segments.filter(s => s.id !== id)`. Clear `drawFuture`. Set `selectedSegmentId` to `null`.

3. `selectSegment: (id) => set({ selectedSegmentId: id })` — simple setter.

4. `setDrawingClick: (p) => { ... }` — This is the FSM. Read `toolMode` and `drawing.clickPoints` from `get()`.
   - If `toolMode === 'line'`:
     - If `clickPoints.length === 0`: set `drawing: { clickPoints: [p], cursorWorld: drawing.cursorWorld }`
     - If `clickPoints.length === 1`: Create LineSegment with `{ id: crypto.randomUUID(), type: 'line', start: clickPoints[0], end: p }`. Call `get().addSegment(lineSeg)` (which also clears drawing).
   - If `toolMode === 'arc'`:
     - If `clickPoints.length < 2`: append p to clickPoints: `set({ drawing: { ...drawing, clickPoints: [...clickPoints, p] } })`
     - If `clickPoints.length === 2`: This is the 3rd click (point-on-arc). Compute circumcircle from `clickPoints[0], clickPoints[1], p`. If `collinear`, set toastMessage to `'Points are collinear — place the arc point off the line'` and reset drawing to `{ clickPoints: [], cursorWorld: null }`. Otherwise, compute startAngle = `Math.atan2(clickPoints[0].y - center.y, clickPoints[0].x - center.x)`, endAngle = `Math.atan2(clickPoints[1].y - center.y, clickPoints[1].x - center.x)`. Compute anticlockwise = `arcDirectionFromThreePoints(clickPoints[0], clickPoints[1], p, center)`. Create ArcSegment with `{ id: crypto.randomUUID(), type: 'arc', center, radius, startAngle, endAngle, anticlockwise, p1: clickPoints[0], p2: clickPoints[1], p3: p }`. Call `get().addSegment(arcSeg)`.

5. `setCursorWorld: (p) => set({ drawing: { ...get().drawing, cursorWorld: p } })` — update cursor position for ghost preview.

6. `clearDrawing: () => set({ drawing: { clickPoints: [], cursorWorld: null } })` — reset transient state.

7. `undoDraw: () => { ... }` — If `drawHistory.length === 0`, return. Pop last from drawHistory into `previous`. Push current `segments` onto drawFuture. Set segments to previous. Also clear selectedSegmentId to null.

8. `redoDraw: () => { ... }` — If `drawFuture.length === 0`, return. Shift first from drawFuture into `next`. Push current `segments` onto end of drawHistory. Set segments to next. Also clear selectedSegmentId to null.

**Modify existing `setToolMode` action:** When switching tool modes, clear drawing transient state by also setting `drawing: { clickPoints: [], cursorWorld: null }` and `selectedSegmentId: null`. This ensures partial click sequences are abandoned on tool switch. Apply this to ALL mode changes (the existing calibrate case and the general case).
