<feature>
  <name>DXF Export Utility</name>
  <files>src/utils/dxfExport.ts, src/utils/dxfExport.test.ts</files>
  <behavior>
    ## isShapeClosed(segments, tolerancePx?)
    - Returns false for empty segments array
    - Returns true when first segment's start endpoint matches last segment's end endpoint within 0.5px (default tolerance)
    - Returns false when first segment's start and last segment's end are further than tolerance apart
    - Works with both LineSegment and ArcSegment types (line uses start/end, arc uses p1/p2)
    - A single segment is NOT closed (start != end for any real segment)

    ## buildDxfString(segments, pxPerMm, unit)
    - Returns a string starting with "0\nSECTION\n2\nHEADER" and ending with "0\nEOF\n"
    - Contains $ACADVER set to AC1009
    - Contains $INSUNITS set to 70\n1 for unit='in', 70\n4 for unit='mm'
    - For a single horizontal line segment at (0,100) to (200,100) with pxPerMm=1, unit='mm':
      - Y-flip: (0,-100) to (200,-100)
      - Origin normalize: (0,0) to (200,0)
      - Scale: 1px = 1mm, so coords are (0,0) to (200,0)
      - DXF LINE entity with group codes 10/20 and 11/21 at these values
    - For an arc segment: center Y is negated, radius is scaled by pxPerMm, angles are converted from radians to degrees with negation, and anticlockwise=true causes start/end angle swap
    - All coordinate values formatted with toFixed(6)
    - Z values (group codes 30, 31) are always 0.000000
    - Arc bounding box includes radius for origin normalization (not just center point)

    ## downloadDxf(content, filename)
    - Creates a Blob with type 'application/dxf'
    - Creates and clicks an anchor element with download attribute set to filename
    - Revokes the object URL after click

    ## generateDxfFilename()
    - Returns "panel-trace-YYYY-MM-DD.dxf" using local date
    - Format: 4-digit year, 2-digit month (zero-padded), 2-digit day (zero-padded)
  </behavior>
  <implementation>
    Implement all functions in src/utils/dxfExport.ts following the exact math from 04-RESEARCH.md:

    1. Helper: `normDeg(d)` -- normalize degrees to [0,360): `((d % 360) + 360) % 360`
    2. Helper: `toRealUnits(worldPx, pxPerMm, unit)` -- `worldPx / pxPerMm`, then `/25.4` if unit==='in'
    3. Helper: `fmt(n)` -- `n.toFixed(6)`
    4. Helper: `getSegStart(seg)` -- returns seg.start for line, seg.p1 for arc
    5. Helper: `getSegEnd(seg)` -- returns seg.end for line, seg.p2 for arc
    6. `isShapeClosed(segments, tolerancePx=0.5)` -- compare getSegStart(first) to getSegEnd(last), return distance <= tolerancePx
    7. `buildDxfString(segments, pxPerMm, unit)`:
       a. Collect all coordinate bounds: for lines, start.x/y and end.x/y; for arcs, center.x +/- radius and center.y +/- radius (arcs extend beyond center)
       b. Apply Y-flip to all Y values (negate)
       c. Compute minX = min of all flipped X coords, minY = min of all flipped Y coords
       d. For each segment: flip Y coords, subtract (minX, minY), scale via toRealUnits, emit DXF entity
       e. For arcs: negate angles (because Y was negated), convert radians to degrees (* 180/PI), normalize to [0,360), swap start/end if anticlockwise===true (canvas CCW becomes CW after Y-flip, swap restores CCW sweep for DXF)
       f. Scale arc radius: toRealUnits(seg.radius, pxPerMm, unit)
       g. Build full DXF string: HEADER section ($ACADVER=AC1009, $INSUNITS=1|4) + ENTITIES section + EOF
    8. `downloadDxf(content, filename)` -- Blob('application/dxf') + anchor click + revokeObjectURL
    9. `generateDxfFilename()` -- local date: `panel-trace-${yyyy}-${mm}-${dd}.dxf`

    IMPORTANT: Do NOT import @tarikjabiri/dxf -- manual DXF string construction per locked decision.
    IMPORTANT: Canvas anticlockwise=true (CCW in screen/Y-down) becomes CW after Y-flip to Y-up. Swap start/end angles so DXF sweeps CCW over the correct arc. Canvas anticlockwise=false (CW in screen) becomes CCW after Y-flip -- emit as-is, no swap needed.
  </implementation>
</feature>
